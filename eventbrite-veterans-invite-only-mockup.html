One template, infinite events. All 4 image assets are uploaded once and reused across every entry-level event. Each new event is a single API call sequence with {City} and {EventDate} swapped in.

1. One-time asset uploads

Upload each PNG once. Store the returned image_id in config. Reuse across every event forever.

Upload sequence (per asset)

  1. POST /media/upload/?type=image-event-logo → returns S3 pre-signed URL + upload token
  2. PUT image bytes to S3 URL → HTTP 204
  3. POST /media/upload/ confirm with token → returns image_id

3. Per-event API sequence

Run these 4 calls in order for every new event. Fully automated.
1Create the event shell

POST /v3/organizations/{ORG_ID}/events/

{
  "event": {
    "name": { "html": "{City} Job Fair {EventDate} - {City} Career Fairs" },
    "start": { "timezone": "{TimezoneId}", "utc": "{StartDateTime}" },
    "end":   { "timezone": "{TimezoneId}", "utc": "{EndDateTime}" },
    "currency": "USD",
    "online_event": true,
    "listed": true,
    "shareable": true,
    "logo_id": "COVER_IMAGE_ID"
  }
}

Returns the new event_id.

2Push the Overview body

POST /v3/events/{event_id}/structured_content/?purpose=listing

Send the modules array. See section 4 for the full payload structure.

3Add FAQ items

For each of the 13 FAQ items: POST /v3/events/{event_id}/questions/

Use Eventbrite's dedicated FAQ component, not the description body. The FAQ component adds schema.org markup which gives an 8% organic traffic boost.

4Publish

POST /v3/events/{event_id}/publish/

Event goes live on Eventbrite.

5. Sanitizer rules — text module

Per Eventbrite's structured_content docs, the text module accepts plain text or HTML. Only the tags below are allowed.

Allowed tags (official)

  • <p> — paragraph
  • <h1>, <h2>, <h3> — headers
  • <b>, <em> — bold & emphasis
  • <a> — link
  • <ul>, <ol>, <li> — lists
  • <aside> — caption
  • <blockquote> — quote

Stripped before render

  • Inline style attributes (no backgrounds, borders, padding, widths, colors, fonts)
  • Class names, IDs
  • <div>, <span> containers (styled or not)
  • <button> — buttons cannot be rendered in text modules
  • <table>, <tr>, <td>, <br>
  • SVG, embedded <img> (images go in their own module)
Implications for our template:
  • The 3-step graphic, the logo grid, the trust stats banner — uploaded as image modules.
  • The employer CTA "Register as an employer →" is wrapped in <h3><a> for prominence — no button styling will survive.
  • Any rules, backgrounds, or borders shown in the mockup are for design preview only — Eventbrite output is flat text + Eventbrite's default tag styles.

6. Native Eventbrite config (outside the description body)

These settings sit on the event object itself, not in structured_content.

Event metadata

  • Refund policy: "Free event — no refund required"
  • Tags: virtual, online, job fair, careers, {city}, entry-level
  • Language: en_US
  • Capacity: large or uncapped (recommend 10,000+)
  • Online event: true

Series

Group all entry-level events under one Eventbrite series. SEO compounds across cities when events are sibling-linked.

Highlights chips

Eventbrite auto-renders these from event metadata. Should show: 4 hours, Online, English.

Ticket class

Single free ticket class:

{
  "name": "Free admission",
  "free": true,
  "quantity_total": 10000
}

7. Cover image specs

  • Recommended size: 2160×1080 (2:1 aspect)
  • Min size: 940×470
  • Format: PNG or JPG
  • Content baked in: title "Entry-Level Virtual Job Fair" + gold "Free for Job Seekers" pill
  • No city name, no date — universal across all 1,000+ events

8. Updating assets later

When we refresh a graphic (e.g., new tile design, updated logo set):

  1. Upload the new PNG → get new image_id
  2. Swap the image_id in config
  3. Future events use the new asset automatically
  4. To update existing live events, run a patch script: POST /events/{id}/structured_content/ with the new payload

Frequently asked questions

FAQ lives in Eventbrite's dedicated FAQ component — NOT in the description body. Each Q/A pair is a separate Eventbrite FAQ item with schema.org markup (the 8% organic traffic boost).
How does MilitaryX work for job seekers?
Register on Eventbrite, then complete your MilitaryX profile. We match your profile against open veterans roles from employers attending this event. If you're matched, we invite you and send you the event location, time, and details.
How do you match job seekers with employers?
Our team reviews your resume, skills, experience, location, and job preferences against open roles from employers attending the event. Strong matches receive invitations.
What happens if I'm not invited?
If your profile doesn't match the open roles at this event, you won't receive an invitation. We'll keep you on file and match you against future MilitaryX events that fit your background.
When do I get the event location?
You'll receive the full event location, time, and details once you're invited. The location is shared with invited candidates only.
Is there a fee to participate?
No. Registering, getting matched, and attending the event is 100% free for job seekers.
What time does the event run?
All MilitaryX events run from 11:00 AM to 3:00 PM local time. Your specific arrival window is shared in your invitation. Plan to stay the full window if possible.
What should I bring?
Bring 4–5 printed copies of your resume, a professional outfit, a notepad and pen, and any portfolio materials relevant to the roles you're interested in.
Can I cancel after I'm invited?
Yes. Cancel as soon as you can so we can offer your spot to another matched candidate. Log into your MilitaryX account or reply to your invitation to cancel.
Can I reschedule?
No, MilitaryX doesn't support job seeker rescheduling. If you can't attend after being invited, cancel as soon as possible. You can be considered for future events that match your profile.
What kinds of employers attend?
Fortune 500 and high-growth employers actively hiring veterans talent in Norfolk. Specific companies vary by event — check your invitation for the attending employer list.
How do I prepare for the event?
Update your JobFairX profile with current skills and experience. Research the attending employers and their open roles. Be ready to discuss your background and interests in short conversations.
What if I'm running late?
Arrive as soon as you can. We'll do our best to fit you in with the employers you've been matched against, but priority goes to candidates who arrive on time.
How long do I need to stay?
Plan for about 2 hours on site. Your matched conversations with employers are typically 10–15 minutes each, plus time to meet additional teams that interest you.
