Developer notes

Post-signup routing: After Step 3 Continue is clicked and the account is successfully created, route the user directly to their employer dashboard at /dashboard. No success/confirmation screen needed — the dashboard's empty state ("You haven't registered for an event") handles next-steps prompting.

Button enable/disable validation:

  • Step 1 (Sign in): Continue button stays disabled until the email field contains a valid email format.
  • Step 2 (Verify email): Create account button stays disabled until exactly 6 digits are entered in the code field.
  • Step 3 (Create account): Continue button stays disabled until all four required fields (Country, Company name, First name, Last name) are filled. Optional fields don't block.

Account creation timing: The user record should be created in the database the moment they verify the email code (end of Step 2), not at the end of Step 3. This lets you flag the record as "incomplete profile" and re-engage by email if they bail before finishing Step 3.

Country default: The Country field on Step 3 should default to United States on initial render. ~95% of accounts are US-based, so defaulting saves a click for almost everyone. Canadian users can switch via the dropdown.
